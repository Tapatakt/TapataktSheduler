# Tapatakt Sheduler

Планировщик задач и дел под Android, построенный на .NET MAUI.

## Статус

**Проект находится в стадии активной разработки и пока не готов к повседневному использованию.**

Реализована основная навигация по экранам, CRUD-операции для дней, типов дней и задач, а также привязка задач к датам и типам дней. Однако ключевые функции для полноценного приложения ещё отсутствуют.

## Что уже работает

- **Календарь** — просмотр месяца, выбор дня, отображение типа дня.
- **Список дел на день** — прямые привязки задач к дате, задачи по типу дня, комментарий к дню, выбор типа дня.
- **Управление задачами** — создание и редактирование задач с временем напоминания по умолчанию и периодичностью повторов.
- **Управление типами дней** — создание, редактирование, удаление типов дней с привязкой задач.
- **Навигация** — Shell с маршрутизацией, переходы между всеми экранами.
- **Визуальный стиль** — единый дизайн, шрифты, цвета, адаптированный под Android.

## Что пока не работает

- **Постоянное хранилище** — все данные хранятся только в памяти. После закрытия приложения дни, задачи, типы дней и привязки теряются.
- **Уведомления и напоминания** — нет фонового сервиса для показа локальных уведомлений в заданное время. Время напоминания сохраняется, но не приводит к реальному оповещению.

## Технологии

- **.NET 10.0** (целевая платформа `net10.0-android`)
- **.NET MAUI** — кроссплатформенный UI-фреймворк
- **CommunityToolkit.Mvvm 8.4.2** — MVVM с source generators (`[ObservableProperty]`, `[RelayCommand]`)
- **XAML Source Generation** (`MauiXamlInflator=SourceGen`)
- **DI** — Singletons для сервисов и `CalendarPage`/`CalendarViewModel`, Transients для остальных ViewModel/Page

## Архитектура

Приложение разделено на отдельные слои:
- `Models` — сущности (`Day`, `DayType`, `PlannedTask`, `TaskTypeBinding`, `TaskDayBinding`, `TaskException`, `TaskCompletion`)
- `Services` — 8 мелких сервисов вместо одного монолита (`DayTypeService`, `DayService`, `TaskService`, `TaskTypeBindingService`, `TaskDayBindingService`, `TaskExceptionService`, `TaskCompletionService`, `SettingsService`), каждый со своим `Lock` и in-memory коллекцией
- `ViewModels` — на `CommunityToolkit.Mvvm`, мгновенное авто-сохранение при редактировании
- `Views` — XAML-страницы с code-behind только для событий UI

## Сборка

```bash
cd TapataktSheduler
dotnet build -f net10.0-android -c Release
```

Для подписанного APK необходим keystore в `Platforms/Android/` и соответствующие настройки в `.csproj`.
